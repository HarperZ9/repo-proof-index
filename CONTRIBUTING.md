# Contributing

Thanks for improving `repo-proof-index`.

## Development

1. Create a branch from `main`.
2. Install test dependencies with `python -m pip install -e ".[test]"`.
3. Run `python -m pytest`.
4. Keep parsers schema-tolerant and add fixtures for new contract shapes.

## Security

Use synthetic proof contracts in issues, examples, and pull requests. Do not
submit private evidence, client names, credentials, or sealed records.
